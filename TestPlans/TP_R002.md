## Acceptance Criteria

1. Continue from Reward Summary on a continuing run goes to Shop, not Combat.
2. All in-between states fire in order: Shop → Formation → Payroll → Combat → Reward → Upkeep → next round's Shop.
3. `SelectedPayrollAction` is null at every Payroll-state entry.
4. Per-hero `Attack`/`UpkeepThisRound` are at base when Round 2's Payroll panel renders.
5. No new shop refresh logic, no new encounters, no scout, no rival. (Offers may refresh per `ChangeState(Shop)` behavior.)
6. End conditions still fire: Morale ≤ 0 → Defeat; Debt ≥ DebtLimit → Defeat; Round 10 win → Victory.

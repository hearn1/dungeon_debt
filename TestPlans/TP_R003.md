# TP_R003 - Fix Hire Placement After Formation Movement

Manual Unity Editor test plan for regression R003.

This plan uses a temporary diagnostic scaffold because `HeroInstance.FormationSlot` is plain C# run state and is not reliably inspectable through Unity Inspector Debug mode.

## Temporary Diagnostic Scaffold

Before running the happy-path and edge-case scenarios, temporarily edit `DungeonDebt/Assets/Scripts/Run/ShopManager.cs`.

In `ShopManager.Hire`, immediately after:

```csharp
offer.Purchased = true;
```

and before:

```csharp
return true;
```

add:

```csharp
string slots = "Hire slots:";
for (int i = 0; i < run.Party.Count; i++)
{
    slots += " " + run.Party[i].Definition.DisplayName + "=" + run.Party[i].FormationSlot;
}
Debug.Log(slots);
```

Look for one Console line per successful hire. Every line should list unique slot values in the range `0` to `4`.
